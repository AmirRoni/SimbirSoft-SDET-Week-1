# UI автотесты для automationteststore.com

Проект выполнен в рамках задания «Практикум SDET: задание UI».

Объект тестирования:
https://automationteststore.com/

## Описание проекта

В проекте реализованы UI-автотесты для интернет-магазина Automation Test Store по чек-листу из задания практикума.

Автотесты покрывают следующие сценарии:
- проверка сортировки товаров в категории
- проверка поисковой выдачи и корзины
- проверка добавления случайных товаров в корзину, удаления части товаров и пересчёта итоговой стоимости

Проект написан на **Python 3.10** с использованием **pytest**, **Selenium WebDriver** и подхода **Page Object Model**.

## Технологии

- Python 3.10
- pytest
- Selenium WebDriver (selenium==4.43.0)
- Chrome (Версия: 147.0.7727.57 (официальная сборка) (64 бит))
- Allure Report
- pytest-xdist

## Что реализовано

### Автотесты
В проекте реализованы следующие тесты:
- проверка сортировки товаров по имени и цене в обе стороны
- поиск по запросу `shirt`, добавление товаров в корзину, изменение количества самого дешёвого товара и проверка итоговой суммы
- добавление 5 случайных товаров, удаление чётных товаров из корзины и проверка итоговой суммы
- базовая smoke-проверка открытия сайта

### Архитектура
Проект организован с использованием **Page Object Model**.

Используются отдельные Page Object-классы для:
- главной страницы
- страницы категории
- страницы поисковой выдачи
- страницы товара
- страницы корзины

### Дополнительно
- оформлены тест-кейсы в Markdown
- подключён Allure Report
- добавлено сохранение скриншота при падении теста
- возможность параллельного запуска тестов

## Покрытие по заданию

### 1. Фильтрация в категориях
Проверяется сортировка товаров:
- по имени по возрастанию
- по имени по убыванию
- по цене по возрастанию
- по цене по убыванию

### 2. Проверка поисковой выдачи и корзины
Проверяется сценарий:
- поиск по запросу `shirt`
- сортировка результатов по наименованию
- добавление 2-го и 3-го товара из выдачи
- использование случайного количества
- переход в корзину
- поиск самого дешёвого товара
- удвоение его количества
- проверка итоговой суммы

### 3. Проверка корзины
Проверяется сценарий:
- добавление в корзину 5 случайных товаров с главной страницы
- использование случайного количества
- удаление всех чётных по порядку товаров из корзины
- проверка итоговой стоимости

## Установка и запуск

### 1. Клонировать репозиторий
```bash
git clone https://github.com/AmirRoni/SimbirSoft-SDET-Week-1.git
cd SimbirSoft-SDET-Week-1
```

### 2. Создать и активировать виртуальное окружение
Windows
```bash
python -m venv .venv
.venv\Scripts\activate
```

macOS / Linux
```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установить зависимости
```bash
pip install -r requirements.txt
```

### 4. Запустить все тесты
последовательный запуск:
```bash
pytest -v
```
параллельный запуск:
```bash
pytest -n 4 -v
```

## Allure Report
### Сгенерировать результаты Allure
последовательный запуск:
```bash
pytest -v --alluredir=allure-results
```
параллельный запуск:
```bash
pytest -n 2 -v --alluredir=allure-results
```

### Открыть отчёт
```bash
allure serve allure-results
```
